H101105 當影像地圖使用為上傳按鈕時，每一作用區域必須分別使用不同的按鈕

不良設計：

下面的例子為影像地圖中包含「搜尋」及「重新輸入」兩個功能按鍵，然而這樣的設計會造成使用者不能分辨不同功能的按鍵。

不良設計：

  <FORM>
  <INPUT type="text" name="keyword1" value="輸入搜尋字串">
  <INPUT name="Submit" type="image" src="images/botton1_2.gif" 
  alt="搜尋與重新輸入按鍵" usemap="#map">
  </FORM>
  <MAP name="map">
  <AREA shape="rect" coords="4,4,54,21" href="#submit" alt="搜尋" title="搜尋">
  <AREA shape="rect" coords="66,3,137,22" href="#clear" alt="重新輸入" title="重新輸入">
  </MAP>

範例結束

正確示範

下面的範例是將影像地圖中的兩個功能按鍵各自拆開，設計成各自獨立的「搜尋」和「重新輸入」兩個圖像按鍵並且分別提供替代文字說明。

正確範例：

  <FORM>
  <INPUT type="text" name="keyword2" value="輸入搜尋字串">
  <INPUT name="Submit" type="image" src="botton1.gif" alt="搜尋">
  <INPUT name="Clear" type="image" src="botton2.gif" alt="重新輸入">
  </FORM>

範例結束
